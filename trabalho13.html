Modularização de Código e Avaliação de LLMs

🐍 Python 🧠 LLMs ⚡ Engenharia Elétrica 📊 Modularização

Desenvolvimento de uma solução para monitoramento de grandezas de engenharia (tensão elétrica), com foco na análise crítica do nível de modularização, legibilidade e qualidade das soluções geradas por diferentes IAs (ChatGPT e Gemini).

🎯 Objetivo do Trabalho

Desenvolver uma solução em Python para monitoramento de grandezas de engenharia (tensão elétrica), com foco na análise crítica do nível de modularização, legibilidade e qualidade das soluções geradas por diferentes LLMs (ChatGPT e Gemini). O trabalho envolve:

  • Desenvolvimento de um sistema de diagnóstico de rede elétrica
  • Validação e filtragem de medições de tensão
  • Classificação de medições em Normal, Alerta e Crítico
  • Detecção de eventos críticos e sequências consecutivas
  • Comparação da modularização entre ChatGPT e Gemini

💻 Código Exemplo (ChatGPT - Estrutura Modular)

# Código gerado pelo ChatGPT
# Sistema de diagnóstico de rede elétrica

import statistics

TENSAO_NOMINAL = 220
TENSAO_MINIMA = 0
TENSAO_MAXIMA = 500

def validar_medicao(valor):
    """Verifica se a medição é numérica e está na faixa válida."""
    try:
        valor = float(valor)
        return TENSAO_MINIMA <= valor <= TENSAO_MAXIMA
    except (ValueError, TypeError):
        return False

def calcular_desvio_percentual(tensao):
    """Calcula o desvio percentual em relação à tensão nominal."""
    return abs(tensao - TENSAO_NOMINAL) / TENSAO_NOMINAL * 100

def classificar_medicao(tensao):
    """Classifica a medição em NORMAL, ALERTA ou CRÍTICO."""
    desvio = calcular_desvio_percentual(tensao)
    if desvio > 20:
        return "CRÍTICO"
    elif desvio > 10:
        return "ALERTA"
    else:
        return "NORMAL"

# Código gerado pelo Gemini
# Sistema de diagnóstico - versão modular

def validar_e_filtrar_medicoes(dados_brutos, faixa_min, faixa_max):
    """Valida e filtra medições, descartando valores inválidos."""
    validas = []
    descartadas = 0
    for valor in dados_brutos:
        try:
            v = float(valor)
            if faixa_min <= v <= faixa_max:
                validas.append(v)
            else:
                descartadas += 1
        except (ValueError, TypeError):
            descartadas += 1
    return validas, descartadas

def calcular_estatisticas(medicoes):
    """Calcula média, mínimo, máximo e desvio padrão."""
    if not medicoes:
        return {"minimo": None, "maximo": None, 
                "media": None, "desvio_padrao": None}
    return {
        "minimo": min(medicoes),
        "maximo": max(medicoes),
        "media": sum(medicoes) / len(medicoes),
        "desvio_padrao": statistics.stdev(medicoes) if len(medicoes) > 1 else 0
    }

📋 Conclusão da Avaliação

1. Os dois são muito bons. Os dois funcionam, são organizados e fáceis de entender.

2. O Gemini ainda ganha por pouco. Ele é um pouquinho melhor porque não usa nenhuma variável global. Tudo entra por parâmetro e sai por retorno. Isso é tecnicamente mais correto.

3. O ChatGPT melhorou muito da primeira versão para essa. A versão 2 está quase no mesmo nível do Gemini. O único "defeito" é que ele ainda usa constantes globais no topo (TENSAO_NOMINAL = 220). Se ele tivesse passado esses valores como parâmetro, teria nota máxima também.

4. Na prática, para um sistema real de monitoramento, os dois códigos seriam aceitáveis. Mas se eu fosse engenheiro e tivesse que escolher um, escolheria o Gemini por ser ligeiramente mais flexível e testável.

💡 O que aprendi: Pequenos detalhes, como evitar variáveis globais, fazem diferença na qualidade do código. Para a faculdade, os dois códigos são ótimos exemplos de modularização.

🧠 Reflexão sobre o Aprendizado

O que aprendi: A análise comparativa entre as soluções geradas pelo ChatGPT e Gemini para o problema de monitoramento de tensão elétrica demonstrou a importância da modularização na qualidade do código. O Gemini se destacou por evitar variáveis globais, tornando o código mais testável e flexível. O ChatGPT também apresentou uma solução de alta qualidade, com estrutura modular e boa legibilidade, perdendo apenas no detalhe das constantes globais. Este trabalho reforçou a importância de avaliar criticamente o código gerado por LLMs e entender os princípios de engenharia de software que fazem a diferença na prática.
